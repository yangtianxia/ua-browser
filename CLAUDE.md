# CLAUDE.md — ua-browser

## Project Overview

`ua-browser` is a zero-dependency TypeScript library for detecting browser, OS, engine, and device type from user agent strings and browser environment variables. It supports browser and Node.js environments.

- **NPM package:** `ua-browser`
- **Current version:** 1.0.0
- **License:** MIT
- **Package manager:** pnpm

## Architecture

### Core design principles

1. **Pure functions, no singletons.** `parseUA(ua, options)` creates no global state. The old singleton class (`const instance = new UaBrowser()`) is gone.
2. **NavContext injection.** All browser environment access (`navigator`, MIME types, language) flows through a `NavContext` parameter. This makes every function testable in Node.js without mocks.
3. **Co-located definitions.** Each browser's detect regex + version pattern + lookup table live together in `BROWSER_DEFS`. Adding a browser is a single-location change.
4. **Explicit priorities.** Detection priority is declared as a numeric `priority` field on each `BrowserDef`. No more implicit reverse-`Object.keys()` ordering.
5. **No sentinel comparisons.** Version extraction uses `RegExp.exec()` + capture groups → returns `null` on no-match. The old `env.version === this.ua` hack is eliminated.

### `src/constants/browsers.ts` — The most important file

Each browser is described by a `BrowserDef`:
```ts
interface BrowserDef {
  name: BrowserName       // the output value
  detect: RegExp          // must match the UA string
  versionPattern: RegExp | RegExp[] | null
  chromeLookup?: Record<string, string>   // Chrome major → display version
  priority: number        // higher = wins when multiple entries match
}
```

When multiple entries match the same UA, the one with the **highest `priority`** wins. App webviews (Wechat, DingTalk, etc.) have the highest priority (500+) to override generic Chrome detection.

**Adding a new browser requires one change here only** (plus updating the `BrowserName` union in `types.ts`).

### `src/utils/extract.ts` — Version extraction

| Function | Purpose |
|----------|---------|
| `extractVersion(ua, pattern)` | Returns first capture group or `null`. Never returns the UA string. |
| `extractVersionFromPatterns(ua, patterns)` | Tries patterns in order, returns first non-null. |
| `lookupFromChromeVersion(ua, table)` | Extracts Chrome major version, maps through lookup table. |

### `src/utils/navigator.ts` — Environment isolation

```ts
interface NavContext {
  userAgent: string
  platform: string
  language: string
  browserLanguage?: string
  maxTouchPoints: number
  mimeTypes?: MimeTypeArray
  connection?: { saveData?: boolean }
  userAgentData?: { ... }
}
```

`getNavContext()` reads the real `navigator` in browser environments, or returns a safe empty object in Node.js. Pass `NavContext` to detectors to make them testable.

### `src/utils/windows-version.ts` — Async Windows 11 detection

```ts
async function getWindowsVersion(nav: NavContext): Promise<string | null>
```

The old code had a module-level floating promise that set a `let _windowVersion` variable — a race condition. The new approach is explicit: callers `await` this function before calling `parseUA()`, then pass the result as `options.windowsVersion`.

### `src/parse.ts` — Assembly

`parseUA(ua, { nav?, windowsVersion? })` runs all four detectors and applies post-detection overrides:
- **360 MIME-type disambiguation** (only runs when `nav` is provided)
- **Baidu/Opera UA override**
- **Chrome generic `\S+Browser` catch-all**
- **Firefox Nightly detection** (only runs when `nav` is provided — browser-only heuristic)
- **Wechat Miniapp detection**

### `src/index.ts` — Public API

```ts
// Default export (call signature from v0.x preserved)
export default function uaBrowser(ua?: string): EnvOption

// Named exports (tree-shakeable)
export { parseUA }
export { isWebview, isWechatMiniapp }
export { getLanguage, getNavContext }
export { getWindowsVersion }
export { VERSION }
export type { EnvOption, BrowserName, OsName, EngineName, DeviceName }
```

## Types

`src/types.ts` is the **sole source of truth** for the public contract. Key types:

```ts
interface EnvOption {
  browser: BrowserName   // e.g. 'Chrome', 'Wechat', 'unknown'
  version: string        // e.g. '124.0.0.0', 'unknown'
  engine: EngineName     // e.g. 'Blink', 'WebKit', 'unknown'
  os: OsName             // e.g. 'Windows', 'iOS', 'unknown'
  osVersion: string      // e.g. '10', '17.4', 'unknown'
  device: DeviceName     // 'Mobile' | 'Tablet' | 'PC'
  isWebview: boolean
  language: string       // e.g. 'zh-CN', 'en-US', 'unknown'
  platform: string       // navigator.platform, e.g. 'Win32', 'MacIntel', 'unknown'
}
```

> **Breaking change from v0.x:** `platfrom` (typo) is now `platform`. The old `EnvPart` and `languge` types are removed.

| File | Format | Notes |
|------|--------|-------|

## Testing

Framework: **Vitest**, running in `node` environment. No browser env needed because all detectors accept injected `NavContext`.

```
test/
├── fixtures/user-agents.ts  ← shared UA strings; add new ones here
├── extract.test.ts          ← unit tests for src/utils/extract.ts
├── browser.test.ts          ← detectBrowser() — one describe per browser family
├── engine.test.ts           ← detectEngine()
├── os.test.ts               ← detectOs()
├── device.test.ts           ← detectDevice()
├── parse.test.ts            ← full parseUA() pipeline, edge cases
└── compat.test.ts           ← API surface regression (no 'platfrom' key, named exports exist)
```

When adding a new browser:
1. Add a UA fixture to `test/fixtures/user-agents.ts`
2. Add a `BrowserDef` entry to `src/constants/browsers.ts`
3. Add the name to the `BrowserName` union in `src/types.ts`
4. Add a test case in `test/browser.test.ts`

## Code Conventions

- **Naming:** camelCase functions, PascalCase types/interfaces, `SCREAMING_SNAKE` for exported constant arrays (`BROWSER_DEFS`)
- **ESLint:** `eslint:recommended` + `@typescript-eslint/recommended` + `import/recommended`. `no-explicit-any` and `no-non-null-assertion` are disabled.
- **TypeScript:** strict mode, `verbatimModuleSyntax`, `moduleResolution: Bundler`, TS 5.x
- **`.js` extensions in imports:** required for ESM compatibility (even though source is `.ts`)
- **No runtime dependencies:** keep it that way
- **`unknown` as default:** all detection returns `'unknown'` when a value cannot be determined, never an empty string or `null`
