# CLAUDE.md — ua-browser

## Project Overview

`ua-browser` is a zero-dependency TypeScript library for detecting browser, OS, engine, and device type from user agent strings and browser environment variables. It supports both browser and Node.js environments.

- **NPM package:** `ua-browser`
- **Current version:** 0.1.9
- **License:** MIT
- **Package manager:** pnpm

## Source Code Architecture

### `src/types.ts`
Defines all public types. This file is **also used as the TypeScript declaration file** for the package — `build.js` copies it verbatim to `dist/index.d.ts`. Keep public types here; do not add implementation code.

Key types:
- `EnvOption` — the full return type of `uaBrowser()`, with typed union fields for `browser`, `os`, `engine`, and `device`
- `OsVersionType` — union of OS names that have OS-version extraction logic
- `HashOptions` — `Record<string, string>` used for Chrome-version-to-named-version lookup tables

Global type augmentations are declared here (`__wxjs_environment`, `chrome.adblock2345`, `Navigator.userAgentData`, etc.).

### `src/browser.ts`
Contains all detection logic. Key elements:

- **Module-level regex maps** (`engineRegExp`, `browserRegExp`, `osRegExp`, `deviceRegExp`, `envRegExp`) — one regex per named value, typed `as const`
- **`hash` object** — pre-computed `Object.keys()` arrays from the regex maps, used for iteration in `getValueOf()`
- **`getValueOf(values, reg, ua)`** — iterates keys in reverse priority order and returns the first regex match
- **`UaBrowser` class** — main detection class
  - `private ua` — stores the active UA string
  - `private version` — object of lazy version-extraction functions keyed by browser name
  - `private osVersion` — object of lazy OS-version-extraction functions keyed by OS name
  - `getEnv(ua?)` — the primary method; returns a full `EnvOption` object
  - `setUA(ua?)` — sets `this.ua` with type validation
  - `getChromeVars()` — extracts major Chrome version, used by lookup-table browsers (360SE/EE, Liebao, etc.)
- **Exported helpers** (used by `index.ts`):
  - `getLanguage()` — reads `navigator.browserLanguage` or `navigator.language`
  - `isWechatMiniapp()` — checks `__wxjs_environment === 'miniprogram'`
  - `isWebview(ua)` — tests for Android webview marker `; wv` in UA

Special detection logic in `getEnv()`:
- 360SE/EE disambiguation via MIME type checks (`chrome.adblock2345`, `application/360softmgrplugin`, etc.)
- iPad on modern macOS detected via `navigator.maxTouchPoints > 1`
- Edge engine: Blink if version > 75, else EdgeHTML
- Chrome/WebKit → Blink upgrade for Chrome > 27, Opera > 12, Yandex
- WeChat browser → WeChat Miniapp upgrade when in mini-program context
- Firefox Nightly detection via `clientInformation` / `u2f` globals

### `src/index.ts`
Thin public wrapper:
1. Creates a singleton `UaBrowser` instance
2. Wraps `instance.getEnv()` in a plain function `Wrap(ua?)`
3. Attaches static helpers (`isWebview`, `isWechatMiniapp`, `getLanguage`, `VERSION`) as properties on `Wrap`
4. Exports `Wrap` as default

## Build System

### Rollup outputs (from `rollup.config.js`)

| File | Format | Minified | Notes |
|------|--------|----------|-------|
| `dist/index.esm.mjs` | ES module | No | Tree-shakeable |
| `dist/index.cjs.js` | CommonJS | No | `exports: 'default'` |
| `dist/index.min.js` | UMD | Yes (Terser) | Global name: `uaBrowser` |

All bundles are transpiled via Babel `@babel/preset-env` targeting IE8+ and iOS 7+.

### `build.js` steps
1. `shx rm -rf dist` — clean previous build
2. `rollup --config rollup.config.js [--watch]` — compile bundles
3. `shx cp src/types.ts dist/index.d.ts` — publish type declarations

### `package.json` exports map
```json
{
  "main": "index.js",          // legacy CJS/UMD fallback
  "module": "./dist/index.esm.mjs",
  "types": "./dist/index.d.ts",
  "exports": {
    ".": {
      "require": "./dist/index.cjs.js",
      "import":  "./dist/index.esm.mjs",
      "types":   "./dist/index.d.ts"
    }
  }
}
```

## Code Conventions

### Naming
- **Classes:** PascalCase (`UaBrowser`)
- **Types/interfaces:** PascalCase (`EnvOption`, `HashOptions`)
- **Functions/methods:** camelCase (`getEnv`, `isWebview`, `getLanguage`)
- **Private methods:** no prefix convention; use `get` prefix for accessors
- **Regex maps:** camelCase + `RegExp` suffix (`browserRegExp`, `osRegExp`)
- **Version/OS lookup tables:** inline objects assigned to `const hash`, keyed by string

### Patterns
- All regex patterns are defined at module scope as `as const` objects — do not inline regexes in logic
- Adding a new browser requires changes in **three places**: `browserRegExp`, `UaBrowser.version`, and the `browser` union type in `types.ts`
- Adding a new OS requires changes in **three places**: `osRegExp`, `UaBrowser.osVersion` (if version extraction needed), and the `os` union type in `types.ts`
- Version extraction uses `.replace()` chains — if the regex doesn't match, `replace` returns the original string, and `getEnv` treats `env.version === this.ua` as `'unknown'`
- `getValueOf()` iterates in **reverse** order — later entries in the regex map have **higher priority**

### TypeScript
- Strict mode is enabled; avoid `any` where possible (though the ESLint rule is disabled)
- `@typescript-eslint/no-non-null-assertion` is disabled — `!` assertions are allowed
- `isolatedModules: true` is set; each file must be a valid standalone module
- The `types.ts` file is the source of truth for declarations — do not maintain a separate `.d.ts` file

### ESLint
- `eslint:recommended` + `@typescript-eslint/recommended` + `import/recommended`
- `camelcase` rule disabled (accommodates browser names like `360SE`)
- `@typescript-eslint/no-explicit-any` disabled
- `no-console` / `no-debugger` are `'off'` (allowed)

## Known Issues / Notes

- There is a **typo in the return type**: `platfrom` (missing 'a') appears in both `EnvPart` type and the returned object — do not fix without a version bump, as it is a breaking API change
- `EnvPart` includes `languge` (typo for `language`) in its type definition, but the actual returned key is `language` from `getEnv()`
- The `test` script references `example/index.js` but the directory was renamed to `docs/` — update the script if the path is confirmed
- No automated test suite exists; consider adding Vitest or Jest for regression coverage
- `_windowVersion` uses `userAgentData.getHighEntropyValues()` asynchronously at module load time — results may not be ready on first synchronous call in some environments
